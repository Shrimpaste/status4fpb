# status4fpb v0.1.0-mvp

## Summary

`v0.1.0-mvp` is the local Web MVP for status4fpb: a QQ group friend status pixel home that stores virtual members, manual statuses, notes, expiration choices, deletion, reset, and recovery entirely in the current browser.

## Included

- Add virtual group friends with display names and simple pixel avatars.
- Set manual statuses, including `套卷中` and `缩圈中`.
- Add short status notes.
- Choose preset expiration times: `不过期`, `30 分钟`, `1 小时`, `2 小时`, or `今天结束前`.
- Fall back to `失联中` after a status expires.
- Delete members with two-click confirmation.
- Reset the local pixel home with two-click confirmation.
- Persist MVP data in browser `localStorage` under `qq-status-pixel-home:v1`.
- Recover safely from corrupt or invalid stored data.
- Document MVP acceptance checks and continuity handoff rules.

## Privacy And Safety

- No QQ private APIs.
- No QQ Bot integration.
- No QQ client scraping.
- No chat monitoring.
- No automatic status inference from private behavior.
- No real QQ IDs stored by default.
- No cookies, tokens, or credentials stored.
- No network sync or transfer for MVP status data.

All member names, notes, and statuses in this MVP are user-entered virtual data.

## Known Limits

- Data is local to the current browser.
- No multi-device sync.
- No import/export flow.
- No automatic QQ status detection.
- No QQ Bot command flow.
- No real-time countdown refresh.
- No desktop tray app yet.
- Pixel-town visuals are still an early MVP shape.
- `今天结束前` uses the user's current browser local timezone.

## Verification

Release verification for this MVP includes:

```bash
npm test
npm run lint
npm run build
```

Browser acceptance should cover:

1. Add a virtual member.
2. Set a status with a note and expiration.
3. Refresh and confirm persisted member, status, note, and expiration.
4. Switch status.
5. Delete the member with two-click confirmation.
6. Add another member.
7. Reset the home with two-click confirmation.
8. Refresh and confirm the home remains empty.
